# Sheet4 按接口分组引脚表

本文档根据 `doc/g3507引脚分配.xlsx` 的 `Sheet4` 整理，并按已确认的 MSPM0 管脚命名规则修正。

整理规则：

- 按 `SPI / I2C / UART / PWM / GPIO / CAN / 电机PWM / Button / 蜂鸣器` 分组。
- `Axx` 统一写作 `PAx`，`Bxx` 统一写作 `PBx`，例如 `A08 -> PA8`、`B09 -> PB9`、`PB00 -> PB0`。
- `TIMA0_C2`、`TIMA1_C0`、`TIMG12_C0` 这类名称表示定时器通道，不是 GPIO 引脚名。
- 原表未明确给出物理引脚的项目保留为 `未标注`。
- 共享总线会在备注中标明，不作为错误冲突处理。
- 冲突处理优先级：`SPI/I2C/UART/CAN/Debug` 最高，编码器次之，`GPIO/PWM/Button/蜂鸣器` 最低。

## 13. 需要注意的资源限制

- `PA26/PA27` 可用 `TIMG8` 做硬件 QEI。
- `PA28/PA31` 对应 `TIMG7`，不能当硬件 QEI 使用，只能用捕获、中断或软件解码。
- 如果两个电机都要求硬件正交编码器解码，当前分配只有一套 `TIMG8 QEI`，另一套需要改为软件解码、外接计数器或重新分配资源。
- SPI0 同时服务 LSM6DSR 和 PMW3901，当前按 PMW3901 最大 SPI 时钟限制配置为 2MHz，两个设备通过 PA17/PA16 手动片选互斥访问。
